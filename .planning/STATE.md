| Duplicate dismissals + merge rules (persistence) | Dismissed duplicate pairs and "always combine these" merge rules are **persisted server-side** (new `DismissedDuplicate` + `DuplicateMergeRule` tables), replacing the old per-browser `localStorage` (which never synced across devices). **Keyed by normalized name, not row id** (`normalizeName().toLowerCase()` for contacts, `normalizeCompanyNameForDedupe()` for companies, pair ordered alphabetically) so a decision survives a **reimport** that assigns new ids.
